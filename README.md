# h5-app-journal

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为 H5 应用或页面内容的集合与版本记录空间。

## 项目简介

`h5-app-journal` 旨在提供一个结构化的方式来存放、管理并展示各类独立的 HTML 页面。这些页面可能包括工具型应用、交互式演示、信息展示页或任何适合以单文件或多文件形式发布的 Web 内容。仓库本身不限定使用场景，适用对象为需要集中托管多个 H5 页面的个人或团队。

## 目录结构

```
h5-app-journal/
├── pages/              # 存放各个独立 HTML 页面及其资源
│   ├── page-001/       # 第一个页面文件夹
│   │   ├── index.html
│   │   └── assets/     # 页面相关资源（CSS、JS、图片等）
│   ├── page-002/
│   │   ├── index.html
│   │   └── assets/
│   └── ...
├── docs/               # 可选的说明文档或笔记
├── README.md           # 本文件
└── .gitignore          # Git 忽略规则
```

建议每个页面独立成文件夹，并在其内部包含 `index.html` 作为入口文件。资源文件统一放置于 `assets/` 子目录，以保持结构清晰。

## 页面归档说明

- **归档范围**：所有页面均为完全自包含或通过相对路径引用资源的 HTML 文件。不依赖外部特定域名服务（除公共 CDN 库外）。
- **版本管理**：每次新增或修改页面时，建议提交对应的 Git commit，并附带简要说明。如需保留历史版本，可保持旧文件不变，通过新文件夹或版本号区分。
- **发布方式**：可通过 GitHub Pages 或其他静态托管服务将仓库根目录或 `pages/` 目录发布为静态站点。具体配置请参考托管平台文档。
